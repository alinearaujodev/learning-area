Para que serve um formulário na web?

Formulário é uma das elementos mais úteis da web, ele está por todas as partes. Para um cadastro simple de login, cadastro uma dado em site, pedir um iFodd... São várias as opções.

E como funciona um formulário? Lembra da lei da atração? Tudo o que vai volta, com os formulários funciona da mesma forma, você enviar algo para uma parte, e depois a outra parte de devolver o que você pediu.

Logo, se queremos ir para um local, precisamos dizer pra onde quero ir, certo? Para isso que serve um atributo chamado action, ele especifica para onde os dados devem ir quando você apertar no botão enviar. Para entender melhor, precisamos estudar um pouco de Cliente e Servidor

Elemento Input e Label

O elemento mais comum no form é o input, que significa entrada, e você pode usa-lo em mais de 20 diferente tipos de entradas, por isso você precisa mudar apenas o atributo type. Veja alguns exemplos abaixo:

O input não possui tag de fechamento. E como mencionei, o atributo mais importante é o type, veja abaixo mais tipos de entrada mais importante:

  • text - para textos
  • password - para senhas
  • color - para selecionar cor
  • number - além de exibir um teclado numérico em dispositivos móveis, nos navegadores modernos exibe um controle que permite incrementar ou decrementar o valor do campo clicando em uma seta para cima ou para baixo. Além dessa diferença visual, é possível determinar valores mínimos, máximos e se há uma escala de valores válidos.
  • submit - botão de enviar
  • email - permite que os dispositivos móveis, principalmente, exiba um teclado adaptado para facilitar esse tipo de entrada. Por exemplo, o iPhone exibe um teclado com o caractere @ e com as opções de domínio .com
  • tel - O input foi especificado para coletar um número de telefone. Em dispositivos com teclados virtuais como smartphones e tablets, é comum o teclado ser adaptado para exibir apenas opções relevantes à entrada de números telefônicos
  • date - para selecionar datas
  • reset - elimina os valores entrados anteriormente nos elementos de um formulário, permitindo que o usuário limpe o mesmo.

Label

Normalmente precisamos de uma descrição para nossos input, por isso é importante a label, esse elemento é uma tag de conteúdo para descrever seu texto. Além disso, é bom para telas pequenas e acessibilidade.

A maneira de conectar uma label para o input é por dois atributos, o for e ID. Todas as vezes que inserimos um input, fornecemos um ID, e todas as que inserimos uma label, fornecemos um for. E quando queremos conecta-los, precisamos atribuir o mesmo valores para ambos.

Outra coisa legal de associar eles, é que assim que clicamos sobre o texto marcado com a tag label, o elemento input é selecionado.

Atributos no Formulários

Atributo Name

Como não é difícil entender o name bem aportuguezado, significa nome, e para o formulário significa apenas uma forma de se referir ao valor do campo que será enviado ao servidor. Geralmente são curtos e não tem espaços.

Por exemplo, se você cria um campo nome do usuario, você logicamente precisa saber o que foi que o usuário digitou naquele, por isso você preciso colocar um atributo name="user", para que assim quando for enviado para o servidor, aparecerá uma coisas parecido com name=aline

Atributo Placeholder

O atributo placeholder exibe o texto contido em seu valor dentro do elemento do formulário caso o seu valor seja vazio.

autofocus Sua presença indica que aquele campo deve iniciar com foco quando a página for carregada. O usuário já pode começar a digitar algo sem nenhum clique. A implementação de autocomplete sem o atributo list no campo, ligando-o a um datalist, vai utilizar os últimos valores utilizados em outros campos ou em outros formulários, dando prioridade a valores adicionados em inputs com o mesmo valor no atributo name.

Se quisermos limitar os números que nossos usuários podem digitar, podemos considerar o uso de um type diferente. Outra opção que poderíamos usar é a configuração type="range" isso vai criar um controle deslizante.

Enviando formulários

Botões

Quando queremos criar um botão que enviam coisas, presicamos conhecer a tag button. Usamos essa tag para envio dentro do form, e sempre será redirencionado para o link do atributo action. O texto do botão passa a ser indicado como conteúdo da tag. Ainda assim é necessário especificar o valor do atributo type, inclusive se ele for button.

O elemento input com o atributo type="button” renderiza um botão dentro do formulário, mas esse botão não tem nenhuma função direta nele e é comumente utilizado para disparar eventos para a execução de scripts. O texto do botão é determinado pelo valor do atributo value.

O elemento input com o atributo type="submit” é similar ao botão, mas quando acionado esse elemento inicia a chamada que envia as informações do formulário para o endereço indicado no atributo action do form
